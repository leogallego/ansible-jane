# Implementation Plan: AAP Remote Control MVP

**Branch**: `001-aap-remote-control` | **Date**: 2026-04-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-aap-remote-control/spec.md`

## Summary

Build a lightweight Android app that connects to an Ansible
Automation Platform (AAP) instance, lets users browse and filter
job templates by name and label, launch jobs with optional extra
variables, and monitor job status via polling. The app uses
Jetpack Compose (Material 3), MVVM with StateFlow, Retrofit for
API communication, and DataStore + Tink for encrypted credential
storage. It auto-detects the AAP API version (2.4 vs 2.5+
gateway) on first connection.

## Technical Context

**Language/Version**: Kotlin (latest stable, targeting JVM 17)
**Primary Dependencies**: Jetpack Compose (Material 3), Retrofit,
KotlinX Serialization, Koin, Jetpack DataStore, Google Tink,
Compose Navigation
**Storage**: Jetpack DataStore + Tink (encrypted, local only)
**Testing**: JUnit 5, Compose UI tests, MockWebServer
**Target Platform**: Android 8.0+ (API 26+)
**Project Type**: mobile-app
**Performance Goals**: Template list loads in <3s, status updates
within 10s, launch in <3 taps
**Constraints**: HTTPS-only (with self-signed cert toggle),
single AAP instance at a time, online-only (no offline mode)
**Scale/Scope**: Single user, ~4 screens, ~100-500 templates max
